題目 1：getProducts()
取得所有產品列表，最基礎的 GET 請求
1
用 fetch() 發 GET 請求到 products 端點
2
第一個 await 等 response，第二個 await 等 .json()
3
API 回傳的資料在 data.products，記得 return
✏️ 填填看
async function getProducts() { const response = fetch( `${BASE_URL}/api/.../products` ); const data = await response.(); return data.; }
✅ 正確答案
async function getProducts() { const response = await fetch( `${BASE_URL}/api/.../products` ); const data = await response.json(); return data.products; }
關鍵記憶
兩個 await 缺一不可：
await fetch() + await response.json()
題目 2：getCart()
取得購物車，回傳格式跟 getProducts 不一樣
這題的差別
URL 結尾換成 /carts
回傳要包成物件：
{ carts, total, finalTotal }
✏️ 填填看
async function getCart() { const response = await fetch( `${BASE_URL}/api/.../` ); const data = await response.json(); return { carts: data., total: data., finalTotal: data., }; }
✅ 正確答案
async function getCart() { const response = await fetch( `${BASE_URL}/api/.../carts` ); const data = await response.json(); return { carts: data.carts, total: data.total, finalTotal: data.finalTotal, }; }
為什麼要包成物件？
呼叫端需要三個值，一起回傳比較方便。這是作業的規定格式。
題目 3：getProductsSafe()
getProducts + 錯誤處理，這題要同時用 response.ok 和 try/catch
1
整個包在 try { } catch (error) { }
2
fetch 完後檢查 response.ok，不成功就 throw new Error()
3
成功回傳 { success: true, data: [...] }
4
失敗回傳 { success: false, error: error.message }
✅ 正確答案
async function getProductsSafe() { try { const response = await fetch(`...`); if (!response.ok) { throw new Error(`HTTP error! status: ${response.status}`); } const data = await response.json(); return { success: true, data: data.products }; } catch (error) { return { success: false, error: error.message }; } }
✏️ 填填看 — 把空格填完
async function getProductsSafe() { try { const response = await fetch(`...`); if (!response.) { throw new (`HTTP error!...`); } const data = await response.json(); return { success: true, data: data.products }; } catch (error) { return { success: , error: error. }; } }
題目 4：addToCart()
POST 請求，三個要素缺一不可
POST 三要素
① method: 'POST'
② headers: { 'Content-Type': 'application/json' }
③ body: JSON.stringify({ data: { ... } })
LiveJS 特有規定
body 要在外面包一層 { data: ... }，不是每個 API 都這樣，看文件決定
✅ 正確答案
async function addToCart(productId, quantity) { const response = await fetch(`...carts`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ data: { productId, quantity } }) }); return await response.json(); }
✏️ 填填看
async function addToCart(productId, quantity) { const response = await fetch(`...carts`, { method: , headers: { 'Content-Type': }, body: ({ data: { productId, quantity } }) }); return await response.json(); }
題目 5：updateCartItem()
PATCH 請求，只改購物車裡的數量
跟 addToCart 的差別
method 換成 'PATCH'
body 換成 { data: { id: cartId, quantity } }
注意是 id（購物車 id），不是 productId
為什麼用 PATCH 不用 PUT？
只改數量這一個欄位，PATCH 只送要改的部分就好，不用整筆重送
✅ 正確答案
async function updateCartItem(cartId, quantity) { const response = await fetch(`...carts`, { method: 'PATCH', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ data: { id: cartId, quantity } }) }); return await response.json(); }
✏️ 填填看 — 跟上題對比著寫
async function updateCartItem(cartId, quantity) { const response = await fetch(`...carts`, { method: , headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ data: { id: , quantity } }) }); return await response.json(); }
題目 6：removeCartItem()
DELETE 單筆，URL 要帶 id
DELETE 的特色
不需要 body
要刪哪一筆，直接寫在 URL：
/carts/${cartId}
✏️ 填填看
async function removeCartItem(cartId) { const response = await fetch( `${BASE_URL}/api/.../carts/`, { method: } ); return await response.json(); }
✅ 正確答案
async function removeCartItem(cartId) { const response = await fetch( `${BASE_URL}/api/.../carts/${cartId}`, { method: 'DELETE' } ); return await response.json(); }
跟 clearCart 的差別
removeCartItem → URL 帶 ${cartId}（刪一筆）
clearCart → URL 不帶 id（刪全部）
題目 7：clearCart()
DELETE 全部，跟 removeCartItem 只差在 URL
這題最簡單
URL 不帶 id → /carts（刪全部）
method: 'DELETE'
不需要 body
✏️ 填填看
async function clearCart() { const response = await fetch( `${BASE_URL}/api/.../`, { method: } ); return await response.json(); }
七題完整對照
GET /products → getProducts
GET /carts → getCart
GET /products +ok → getProductsSafe
POST /carts → addToCart
PATCH /carts → updateCartItem
DELETE /carts/:id → removeCartItem
DELETE /carts → clearCart
常見錯誤 & 怎麼 Debug
卡住的時候，先看這裡
❌ 拿到 undefined
可能原因：
1. response.json() 漏了 await
2. 函式裡沒有 return
3. data.products 路徑寫錯

Debug 方法：一層層加 console.log
❌ POST 沒效果
1. 忘了加 Content-Type header
2. body 沒有用 JSON.stringify()
3. LiveJS 忘記包 { data: ... }
✅ Debug 三步驟
console.log(response);
// ← fetch 回了什麼？
console.log(data);
// ← parse 完是什麼結構？
console.log(data.products);
// ← 路徑對不對？
Promise pending 還在？
看到 Promise { <pending> }
→ 某個地方漏了 await
→ 從第一個 await 往下查
實戰總結
三個 pattern 記住，七個函式全部搞定
Pattern 1：GET
await fetch(url)
await response.json()
return data.xxx

→ getProducts / getCart
Pattern 2：GET + 錯誤處理
try { ... } catch (e) { }
if (!response.ok) throw Error
return { success, data / error }

→ getProductsSafe
Pattern 3：POST / PATCH
fetch(url, {
method: 'POST'/'PATCH',
headers: { Content-Type },
body: JSON.stringify({data})
})
→ addToCart / updateCartItem
Pattern 4：DELETE
fetch(url 或 url/id, {
method: 'DELETE'
})
不需要 body

→ removeCartItem / clearCart
一句話記住
GET 兩個 await → POST 三要素 → PATCH/DELETE 換方法就好
