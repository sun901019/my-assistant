LINE 每天自動收到早報與晚報，不用手動發送。整個流程是：懂名詞 → 建 Channel → 拿 Token & User ID → 設定 .env → 測試並排程。
課前條件
需要 Windows 10 / 11、基本 PowerShell 操作能力，以及可登入的 LINE 帳號。
今天要把 LINE 每日推播系統完整設定起來。最後的成果是每天早上 8 點、晚上 8 點自動收到推播。

Access Token 長這樣
一串很長的英數字，從 LINE Developers 後台「發行」取得。
LINE_ACCESS_TOKEN=xxx...xxx
User ID 長這樣
固定以大寫 U 開頭，後面跟 32 位英數字。這是你自己的 LINE 識別碼。
U1234abcd...（共 33 碼）
常見誤解
LINE Notify 是已停用的舊服務。本系統使用的是 LINE Messaging API，兩者完全不同。
這一頁的重點是把 LINE Notify 和 LINE Messaging API 分清楚，後面操作才不會跟舊文件混在一起。
在 LINE Developers 建立 Channel
1
開啟瀏覽器，進入 developers.line.me，用你的 LINE 帳號登入。
2
點 Create a new provider，輸入任意名稱（例如：My Assistant）後建立。
3
點 Create a new channel，選 Messaging API。
4
填寫頻道名稱與說明，同意條款後建立。
5
進入 Channel 的 Messaging API 分頁，找到 Channel access token，按 Issue 並複製 Token。
6
在同一分頁找到 Bot basic ID，用 LINE 掃描 QR Code 或搜尋 ID，把機器人加為好友。
範例 Token
LINE_ACCESS_TOKEN=Abcd1234EfghIjkl5678MnopQrst9012UvwxYzab...（共約 172 個字元）
完成標誌
你已複製一串很長的 Token，並在 LINE 裡看到機器人帳號好友。Token 請立即複製，視窗關掉後就看不到了。
最容易漏掉的兩件事：Issue Token 後要立刻複製，以及機器人一定要加好友。
取得你的 User ID（最常卡這步）
方法 A：自架 Webhook
1. 在 LINE Developers → Channel → Messaging API 開啟 Use webhooks
2. 到 webhook.site 取得臨時 URL，貼進 Webhook URL
3. 用你的 LINE 傳任何訊息給機器人
4. 在 webhook.site 找到 source.userId 並複製其值
方法 B：請 AI 幫你（推薦）
適合不熟 webhook 設定、只想快速完成的使用者。把下面提示語貼給 Claude、ChatGPT 或任何 AI 助理即可。
AI 提示語
我在設定 LINE Messaging API 推播系統，需要取得我自己的 LINE User ID（以大寫 U 開頭的 33 碼字串）。我已經在 LINE Developers 建好一個 Messaging API Channel，也把機器人加為好友了。請告訴我最簡單的方法取得 User ID，我使用 Windows，不熟悉架設 webhook 伺服器。
完成後你有
一個以 U 開頭的 33 碼字串。注意：webhook.site 的 URL 是臨時的，只需要用一次取得 User ID 就夠了。
User ID 不會直接顯示在後台，必須讓機器人接到你傳的訊息，才能知道你的 ID。
把 Token 和 User ID 填進 .env
先在 PowerShell 複製 .env.example：
PowerShell
cd C:\path\to\my-assistant\line-notify Copy-Item .env.example .env
LINE_USER_ID（U 開頭的 33 碼）
LINE_ACCESS_TOKEN（Channel access token）
格式說明
用文字編輯器開啟 .env 後，直接填值，不要加引號。
# 在下方填入你的資料，不要加引號 LINE_USER_ID=（等待輸入） LINE_ACCESS_TOKEN=（等待輸入）
常見格式錯誤
不要在 = 後加引號；變數名稱必須是 LINE_ACCESS_TOKEN，不是 LINE_CHANNEL_ACCESS_TOKEN。這個檔案不要截圖、不要貼到聊天室、不要上傳 GitHub。
這一頁保留即時驗證器，讓使用者直接檢查 User ID 和 Token 格式。

發晚報
cd C:\path\to\my-assistant\line-notify ..\.venv\Scripts\python.exe .\digest.py --mode evening

驗收清單（勾完才算完成）
如果有顯示成功但 LINE 沒收到，優先檢查機器人是否已加好友，以及 User ID 是否填成自己的 source.userId。
每天改這一個檔案：schedule.json
events（行程）
1
date 格式：YYYY-MM-DD（例如 2026-05-07）
2
priority：high = 紅色重要提醒；normal = 一般提醒
3
done 完成後改成 true，不會再顯示在早報
tasks（待辦）
1
status：todo / in_progress / done
2
done_at：完成日期 YYYY-MM-DD；未完成填 null
行程標題
日期（YYYY-MM-DD）
時間（HH:MM）
重要度
日常使用
設定完成後，日常主要就是改這個 JSON 檔案。若不確定格式正確，可用 python -m json.tool 驗證。
這一頁保留 JSON 生成器，讓使用者快速產生格式正確的行程片段。
Windows 工作排程器設定
先決條件
手動推播必須先成功，再來設定自動排程。完成後，電腦每天 08:00 和 20:00 自動推播，不用手動。
先填入你的專案路徑（後面指令會自動替換）
例如：C:\Users\SunLee\projects\my-assistant
方法 A：PowerShell 一鍵建立（推薦）
系統管理員 PowerShell
$Project = "C:\path\to\my-assistant" $Python = "$Project\.venv\Scripts\python.exe" $Script = "$Project\line-notify\digest.py" $WorkDir = "$Project\line-notify" $MornA = New-ScheduledTaskAction -Execute $Python -Argument "`"$Script`" --mode morning" -WorkingDirectory $WorkDir $MornT = New-ScheduledTaskTrigger -Daily -At 8:00AM Register-ScheduledTask -TaskName "line-notify-morning" -Action $MornA -Trigger $MornT $EveA = New-ScheduledTaskAction -Execute $Python -Argument "`"$Script`" --mode evening" -WorkingDirectory $WorkDir $EveT = New-ScheduledTaskTrigger -Daily -At 8:00PM Register-ScheduledTask -TaskName "line-notify-evening" -Action $EveA -Trigger $EveT
測試排程
Start-ScheduledTask -TaskName "line-notify-morning"
方法 B：工作排程器 GUI
1
開啟「工作排程器」，點「建立基本工作」，名稱填 line-notify-morning
2
觸發程序選「每天」，時間設 08:00
3
程式填 C:\path\to\my-assistant\.venv\Scripts\python.exe
4
新增引數填 C:\path\to\my-assistant\line-notify\digest.py --mode morning
5
開始位置填 C:\path\to\my-assistant\line-notify
6
再建一個 line-notify-evening，時間改 20:00，引數改 --mode evening
PowerShell 方式最快，GUI 方式則適合不想碰指令的人。兩種做法都保留。
卡關了？對照這裡
沒收到LINE 沒收到訊息，但指令沒報錯▶
確認機器人帳號已加為好友。確認 LINE_USER_ID 是 source.userId，不是 channel id 或 bot id。
401LINE 回傳 401 Unauthorized▶
Token 錯誤或過期。回 LINE Developers → Messaging API → 重新發行 Channel access token → 更新 .env 的 LINE_ACCESS_TOKEN。
400LINE 回傳 400 Bad Request▶
User ID 格式不對或機器人帳號未加好友。確認 LINE_USER_ID 以大寫 U 開頭、共 33 碼。
缺模組ModuleNotFoundError: No module named 'httpx'▶
cd C:\path\to\my-assistant .\.venv\Scripts\python.exe -m pip install httpx
JSONjson.decoder.JSONDecodeError▶
schedule.json 格式壞掉了，執行：
..\.venv\Scripts\python.exe -m json.tool .\schedule.json
常見錯誤：最後一筆後多了逗號，或把 null / true / false 加了引號。
排程失敗手動成功，但工作排程器沒有發▶
檢查：① 程式必須是 .venv\Scripts\python.exe 完整路徑 ② 開始位置必須是 line-notify 資料夾 ③ 電腦在排程時間必須開機
Get-ScheduledTaskInfo -TaskName "line-notify-morning"
LastTaskResult 不是 0 代表上次執行失敗。
路徑路徑包含空白（如 C:\Users\Sun Lee）▶
PowerShell 用引號包住完整路徑：
& "C:\Users\Sun Lee\projects\my-assistant\.venv\Scripts\python.exe" "C:\Users\Sun Lee\projects\my-assistant\line-notify\digest.py" --mode morning
缺變數缺少環境變數：LINE_USER_ID, LINE_ACCESS_TOKEN▶
.env 放錯位置，或變數名稱打錯。確認：① 檔案路徑是 line-notify\.env ② 變數名稱是 LINE_ACCESS_TOKEN ③ = 前後沒有多餘空格或引號。
