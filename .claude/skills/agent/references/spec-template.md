# SPEC 模板 — 新 Skill 設計規格書

建立複雜 skill 前，先寫這份 SPEC 讓老闆確認，確認後才產出 SKILL.md。

---

## 1. 核心定位

```

```

## 2. 邊界設定

```
✅ 適用情況：
❌ 不適用情況：
⚠️ 跟哪些 skill 容易混淆，差異在哪：
```

## 3. 觸發條件

```
核心觸發詞：
延伸情境（也該觸發）：
不該觸發的邊界情境：
```

## 5. 工作流程

每步驟都要寫清楚：做什麼、確認什麼、輸出什麼。

```
Step 1:
  做：
  確認：
  輸出：

Step 2:
  做：
  確認：
  輸出：
```

```

例外情況 B：→ 處理方式
```


## 9. 參考資源

```

```

## 10. 完成度檢查清單

```
- [ ] 名稱清晰、觸發可測
- [ ] 流程每步有檢查點
- [ ] 範例完整
- [ ] 邊界清楚不跟其他 skill 衝突
- [ ] SKILL.md 行數 < 200（細節放 references/）
```
